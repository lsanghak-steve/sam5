## 1. 게임 콘셉트

삼국지5 파워업키드의 감각을 참고한 턴제 군주 전략 시뮬레이션 게임을 만든다. 플레이어는 군주가 되어 도시를 다스리고, 장수를 등용하며, 외교와 전쟁을 통해 세력을 확장한다.

직접적인 복제보다는 다음 요소를 현대적인 웹/앱 형태로 재해석한다.

- 여러 세력이 도시를 차지한 상태에서 시작하는 시나리오
- 도시별 내정, 병력, 금, 군량 관리
- 장수의 능력치와 충성도, 소속 관리
- 턴 단위 명령 실행
- 도시 간 이동과 전쟁
- 간단한 이벤트와 AI 세력 행동

## 2. 목표 플랫폼

초기 버전은 로컬에서 바로 실행 가능한 웹 게임으로 만든다.

- 첫 목표: 브라우저 기반 싱글 플레이 프로토타입
- 우선순위: 빠르게 플레이 가능한 코어 루프 완성
- 저장 방식: 초기에는 브라우저 메모리 또는 localStorage

## 3. 핵심 플레이 루프

1. 시나리오를 선택한다.
2. 플레이어 세력을 선택한다.
3. 도시 화면에서 내정, 징병, 훈련, 등용, 이동, 전쟁 명령을 선택한다.
4. 턴을 종료하면 플레이어 명령과 AI 세력 행동이 처리된다.
5. 도시 지배, 장수 상태, 자원 변화가 반영된다.
6. 모든 도시를 통일하거나 플레이어 세력이 멸망하면 게임이 종료된다.

## 8. 1단계 시작 기록

### 2026-05-24 1단계: 프로젝트 기획서 작성

기능과 내용을 구현하기 전에 우선 `plan.md`를 생성했다. 이 문서는 게임의 방향, 핵심 시스템, 데이터 모델, UI 구조, 기능 확인 기준을 정의한다. 다음 단계에서는 현재 저장소 구조를 확인한 뒤, 로컬 웹 게임 프로젝트를 구성하고 최소 실행 화면을 만든다.

### 2026-05-24 2단계: 정적 웹 프로토타입 구성

이번 단계에서는 별도 패키지 설치 없이 브라우저에서 바로 실행되는 정적 웹 게임을 만든다. `index.html`, `styles.css`, `app.js`를 생성하고, 초기 시나리오 데이터와 턴제 명령 처리 로직을 포함한다. 목표 기능은 도시 선택, 내정/군사 명령, 턴 종료, 간단한 AI 행동, 자동 전투, 승패 판정까지 한 화면에서 확인할 수 있는 최소 플레이 가능 버전이다.

### 2026-05-24 3단계: 기능 확인

프로토타입 파일 생성 후 `app.js` 문법 확인과 로컬 브라우저 실행 확인을 진행한다. 확인할 기능은 세력 선택, 도시 선택, 명령 버튼 활성화, 내정/군사 명령 처리, 공격 대상 선택, 턴 종료 후 AI 행동 로그 출력, 저장 버튼 동작이다.

### 2026-05-24 3단계 보강: 로컬 실행 서버 추가

브라우저 보안 정책상 `file://` 직접 실행이 제한될 수 있으므로, 의존성 없이 Node 내장 모듈만 사용하는 `server.js`를 추가한다. 이 서버는 현재 폴더의 정적 파일을 `http://127.0.0.1:5173`에서 제공하며, 개발 중 화면 확인에 사용한다.

### 2026-05-24 3단계 보정: 사건 로그 조사 처리

자동 검증 중 전투 결과 로그에서 세력명 뒤 조사가 어색하게 출력되는 것을 확인했다. 한국어 이름의 받침 여부에 따라 `이/가`를 고르는 작은 유틸리티를 추가하고, AI 행동과 점령 로그 문장을 자연스럽게 보정한다.

### 2026-05-24 3단계 확인 결과

`app.js`와 `server.js`의 문법 검사를 통과했다. 자동 검증으로 유비 세력 선택, 수춘 개발 명령, 턴 종료 후 190년 2월 진행, 명령 버튼 6개와 도시 버튼 8개 렌더링, 저장 버튼 활성화를 확인했다. 전투 검증에서는 수춘에서 허창을 공격해 소유권이 유비로 바뀌고, 사건 로그가 `유비가 조조의 허창을 점령했습니다.`처럼 자연스럽게 출력되는 것을 확인했다. 단, Codex 내장 브라우저가 현재 로컬 주소 접근을 차단해 실제 시각 확인은 사용자가 브라우저에서 `http://127.0.0.1:5173`을 열어 확인해야 한다.

### 2026-05-24 4단계: 로컬 서버 연결 문제 확인

사용자 브라우저에서 `127.0.0.1` 연결 거부 화면이 표시되었다. 이는 보통 정적 서버가 실행 중이 아니거나 실행 직후 종료되었을 때 발생한다. 이번 단계에서는 5173 포트 상태, Node 서버 실행 상태, 로컬 응답 여부를 확인하고, 필요하면 실행 방법을 더 안정적으로 보강한다.

### 2026-05-24 4단계 보강: 더블클릭 실행 파일 추가

확인 결과 `node server.js`를 터미널에서 직접 실행하면 서버는 정상적으로 시작된다. 연결 거부 원인은 브라우저를 열 때 서버가 실행 중이 아니었기 때문이다. 사용자가 매번 명령어를 기억하지 않아도 되도록 `start-game.bat`을 추가한다. 이 파일은 프로젝트 폴더로 이동한 뒤 브라우저를 열고 `server.js`를 실행한다.

### 2026-05-24 4단계 수정: Windows 정적 파일 경로 처리 보정

별도 터미널에서 서버를 실행한 뒤 연결 거부는 해결되었지만 루트 주소 `/` 요청이 404를 반환했다. Windows에서 `/`를 `path.normalize`로 처리할 때 루트 경로로 해석될 수 있어 `index.html` 매핑이 실패한 것이다. 요청 경로의 앞쪽 슬래시를 제거하고 `path.resolve`로 프로젝트 폴더 내부 파일만 제공하도록 `server.js`를 수정한다.

### 2026-05-24 4단계 확인 결과

수정된 서버를 새 터미널에서 다시 실행했다. `netstat`로 `127.0.0.1:5173`이 `LISTENING` 상태임을 확인했고, `http://127.0.0.1:5173`과 `http://127.0.0.1:5173/app.js` 요청이 모두 HTTP 200으로 응답하는 것을 확인했다. 이후 기본 브라우저로 게임 주소를 열었다.

### 2026-05-24 5단계: 명령 진행 표시 추가

현재 명령은 버튼을 누르는 즉시 결과가 반영되어 진행 중이라는 느낌이 부족하다. 이번 단계에서는 명령을 선택하면 진행 상태 패널이 나타나고, 진행률 바와 단계 문구가 짧게 움직인 뒤 실제 결과가 반영되도록 수정한다. 진행 중에는 중복 명령과 턴 종료를 막고, 완료 후 사건 기록과 도시 수치가 갱신되게 한다.

### 2026-05-24 5단계 확인 결과

명령 패널에 진행 상태 영역을 추가했고, 내정/군사 명령과 출진, 턴 종료가 진행률 표시 후 완료되도록 수정했다. 자동 검증에서 유비 선택 후 수춘 개발 명령을 실행했을 때 진행 상태가 활성화되고 턴 종료 버튼이 잠겼으며, 완료 후 개발도가 48에서 55로 상승하고 수춘이 행동 완료 상태가 되는 것을 확인했다. 실행 중인 로컬 서버도 `index.html`과 `app.js`를 HTTP 200으로 제공하는 것을 확인했다.

### 2026-05-24 6단계: 명령 가능 횟수 표시

현재 게임 규칙은 도시 1곳당 한 달에 명령 1회를 수행할 수 있는 구조다. 사용자가 남은 명령 횟수를 쉽게 알 수 있도록 명령 패널에 이번 달 명령 가능 횟수, 전체 보유 도시 수, 이미 명령을 수행한 도시 수, 선택한 도시의 명령 가능 여부를 표시한다. 명령 완료와 턴 종료 후 표시가 즉시 갱신되도록 한다.

### 2026-05-24 6단계 확인 결과

명령 패널에 `이번 달 명령 n/m회 남음` 표시를 추가했다. 자동 검증에서 유비 선택 직후 `이번 달 명령 3/3회 남음`, 수춘 개발 명령 완료 후 `이번 달 명령 2/3회 남음`으로 갱신되는 것을 확인했다. 선택 도시 상태도 `수춘에서 명령을 내릴 수 있습니다.`에서 `수춘은 이번 달 명령을 완료했습니다.`로 바뀌는 것을 확인했다.

### 2026-05-24 7단계: 지도와 영토 확장

현재 지도는 8개 도시 중심이라 통일 전략 게임으로는 공간감이 부족하다. 이번 단계에서는 서북, 관중, 형주, 강남, 남방 도시를 추가하고 세력도 늘려 더 넓은 땅을 보여준다. 목표는 도시 수를 16개 이상으로 확장하고, 도시 간 연결을 늘려 여러 방향으로 진출할 수 있게 만드는 것이다. 명령 가능 횟수 표시는 보유 도시 수에 따라 자동으로 늘어나므로 확장된 지도에서도 같이 갱신된다.

### 2026-05-24 7단계 확인 결과

도시를 17개, 세력을 8개로 확장했다. 새로 추가한 주요 도시는 낙양, 장안, 천수, 한중, 양양, 무릉, 영릉, 회계, 교지이며, 동탁, 마등, 유표, 사섭 세력을 추가했다. 자동 검증에서 지도 도시 버튼이 17개 생성되고, 유비 세력의 보유 도시가 4개라 명령 가능 횟수가 `4/4회 남음`으로 표시되는 것을 확인했다. 모든 인접 도시 연결 ID가 실제 도시를 가리키고 지도 경로가 생성되는 것도 확인했다.

### 2026-05-24 8단계: 금, 군량, 병력 수정 기능

테스트와 밸런스 조정을 쉽게 하기 위해 선택한 도시 기준의 간단한 편집 기능을 추가한다. 금과 군량은 선택 도시를 소유한 세력의 자원으로 수정하고, 병력은 선택 도시의 병력으로 수정한다. 입력값은 음수가 되지 않게 제한하고, 수정 후 도시 정보와 사건 기록이 즉시 갱신되도록 한다. 진행 중인 명령이 있을 때는 편집을 막아 상태 꼬임을 방지한다.

### 2026-05-24 8단계 확인 결과

도시 정보 패널에 자원 수정 영역을 추가했다. 선택한 도시 기준으로 금과 군량은 해당 도시 소유 세력의 자원으로, 병력은 선택 도시의 병력으로 수정된다. 자동 검증에서 수춘 선택 후 금 999, 군량 2222, 병력 777을 적용했을 때 상태와 도시 정보가 즉시 갱신되고 `수춘의 자원을 수정했습니다.` 로그가 남는 것을 확인했다. 음수 입력은 0으로 제한되는 것도 확인했다. 로컬 서버는 `index.html`과 `app.js`를 HTTP 200으로 제공 중이다.

### 2026-05-24 9단계: 장수 포상 기능
